It looks like I learned a lot but a lot of this stuff is overlapping (or rather compounding knowledge rather than an entirely new topic.) For one, the width and height are self explanatory, they allow you to alter the width and height of the content box that naturally surrounds HTML elements. I bet you're wondering then, what exactly is the content box? Well, that's defined through block-level elements, inline-elements, and inline-block elements. Then margin and padding work based on those things too! Margin adds spacing outside of the content box, while padding adds space in it. The CSS combinators on the other hand are just ways to reference html elements in CSS based on relative info, like siblings and parents for instance.

Content Box

Since I keep referencing this word, I will just give a quick section demonstrating it. The content box is the borders surrounding an element, and there are two types. The block-level elements and the inline-elements. The block-level elements take up an entire line of space, no matter the inner content.

CSS Combinators

CSS combinators are a way to select HTML elements based on relative siblings and parents

/* Descendant */
.parent .child {
}

/* Child */
.parent > .child {
}

/* Adjacent Sibling */
.first + .second {
}

/* General Sibling */
.first ~ .second {
}

Child is different from descendant since it only gets the immediate child. Meaning that it will get the first child and no others (specifically children that are nested within another parent, such as a div).

In the same way, general sibling is different than adjacent sibling. General sibling gets every matching sibling after the first element, whereas adjacent sibling only gets the first.
