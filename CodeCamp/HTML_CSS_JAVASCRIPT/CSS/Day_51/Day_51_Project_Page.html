Inheritance

Inheritance is (I'd imagine) for allowing programmers not to have to rewrite their CSS every single time for every single element. Essentially, it allows for certain CSS declerations to be inherited from parent element to child element. However, some properties are not inheritable unless specified thorugh the "inherit" value with the specified property. Here's an example

Ancestry

I am the parent. My color is blue

I am the child, I inherited the color!

I am the parent. My padding is 20px

I am child A. My padding is NOT 20px because that is not a naturally inheritable property, and I have nothing altering that.

I am child B. My padding IS 20px because I have altered the inheritability of padding by using padding: inherit;

Specificity

Specificity is how CSS decides which declerations matter more than others. It's apart of a much larger algorithm (the cascade algorithm) which is a combination of a lot of different things to determine which CSS is displayed on a webpage. (Since remember, there can be user inputted CSS, your CSS, the browser's CSS, there is a LOT of CSS).

Link States

Link states are the different states a link has. There are 5 different states (that you'll use the most.)

a:link
An unvisited link
a:visited
A visited link
a:hover
When the mouse is over the link
a:focus
When the element has keyboard focus (like pressing tab)
a:active
While you're actively clicking/holding the mouse button on it
